3. Chains:

Used to build pipeline in langchain.
For example we need to build a LLM application which takes english text let say of 1000 words and convert it to hindi summary of let say 100 words.

Now Flow:

Query----> LLM----> translate(hindi)----> LLM----> summarize(100 words)---->output
Now we need to build this pipeline.
Without langchain we need to write code for each component and then integrate them.
With langchain we can build this pipeline in few lines of code.
And langchain provides a way through which these components can be integrated automatically.
i.e it will take the output of one component as input of another component automatically.

Can form complex pipelines using these components as well.

Complex Pipeline:
1. Parallel pipeline

Input.  ----> LLM1 ---->
                        report--------> LLM ---> combine report.
       ------>LLM2----> 

2.Conditional pipeline

input----> feedback----> good----> thankyou
                   ----> bad----> email(customer support service).

4. Indexes:

---> Document loader
---> Text splitter---> data chunking.
---> Vector Store---> so that semantic search can be performed.
---> Retriever---> user query--> embedding---> semantic search---> relevant documents.

(Relevant documents + user query)----> LLM----> output.

For example we are asking our personal question or releated to company documents.
But normal LLM is trained on internet and it doesn't have any idea about company documents.
So here if we combine LLM with indexes then it can provide better results.

i.e LLM + external knowledge.

5. Memory:

LLM Api calls are stateless.
---> i.e. each request to LLM api is independent of previous requests.

It means if we want to remember previous conversation to provide context to the LLM for current conversation etc we have to provide the memory to the LLM.

ConversationBufferMemory
ConversationBufferWindowMemory
Summarizer-Based Memory
Custom Memory

next req---> (query + memory(previous conversation history))----> LLM----> output.
